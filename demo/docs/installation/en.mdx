Install Scribekit into an existing Next.js App Router app and import its stylesheet. You need
Next 14 or newer and React 18 or newer.

## Install the package

```bash
npm install @daanvandenbergh/scribekit
```

Scribekit renders MDX with `next-mdx-remote`, which is an **optional** peer dependency, so npm
will not install it for you. Add it explicitly:

```bash
npm install next-mdx-remote
```

Skip this and the package installs cleanly, then fails at your first docs page with a
module-not-found for `next-mdx-remote/rsc`.

## Check the peer dependencies

You almost certainly already have the first three. In practice only `next-mdx-remote` needs adding.

| Package | Range | Notes |
| --- | --- | --- |
| `react` | `>=18` | required |
| `next` | `>=14` | the package does not work without it |
| `react-dom` | `>=18` | optional peer |
| `next-mdx-remote` | `^5 \|\| ^6` | optional peer, install it yourself |

The optional peers produce no npm warning when they are missing, so a gap surfaces as a build
error rather than an install error.

## Import the stylesheet

Import the bundled stylesheet once, app-wide (for example in your root layout):

```tsx
import "@daanvandenbergh/scribekit/styles.css";
```

Colours, fonts, shadows, and widths read from `--scribekit-*` custom properties with inline
fallbacks, so you can restyle the components without touching them. See [Theming](/docs/theming).
